# Environment variables for the Guidewire suite. NAMES ONLY — never values (some secret).
#
# Tags:
#   [set]        required, non-sensitive — skill can help set (known/derivable values)
#   [req]        required — you set it yourself (e.g. DEPLOYMENT_ID)
#   [opt]        optional — only needed to hit LIVE integration servers (else mock them)
#   [opt-secret] optional AND secret — only for live integrations
#
# The checker NEVER prints values (they stay masked in chat); it only reports set / missing.
#
# DEPLOYMENT_ID: swap the environment segment (e.g. the "qa3" in ...:dev:qa3:/deployment/...)
# to whatever env you want to mirror (dev2, etc.). Check/change its value in your own shell.
#
# The optional vars are NOT required to get the suite running — set the integrations to
# mock if you don't need live servers. Provide them only to hit live integrations. Never
# store or commit their values.
#
# (Run the server in the local env via -Denv=local — a JVM property in the gwb command /
#  Studio run config, not an exported env var.)
